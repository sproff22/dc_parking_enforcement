# DC Parking Enforcement Explorer — UI

Interactive web dashboard for exploring parking citation risk and enforcement equity across Washington, D.C.'s eight wards.

## Features

- **Interactive Ward Map**: Choropleth map showing enforcement risk by ward, colored by Bayesian recurrence scores
- **Address Search**: Enter any DC address to get a personalized A–F enforcement grade for your specific block
- **Spatiotemporal Filtering**: Filter by violation type, time of day, season, and day of week to see how risk changes
- **Equity Analysis**: Examine correlations between citation patterns and demographics (income, poverty, racial composition)
- **Real-Time Statistics**: View ward-level citations, fine revenue, top violations, and hourly/monthly patterns
- **Per-Capita Toggle**: Normalize citation counts by population to account for ward size differences

## Technology Stack

- **React 18** — UI framework
- **Vite 5** — Build tool and dev server
- **Recharts 2** — Charting library for scatter plots and time series
- **D3-like manual SVG** — Custom choropleth map rendering
- **Client-side only** — No backend required, all data pre-computed and bundled as JSON

## Running the Application

The UI is built with Vite + React and runs entirely in the browser:

```bash
cd ui
npm install        # Install dependencies (requires Node.js 16+)
npm run dev        # Start development server
```

The development server will start at `http://localhost:5173` and automatically reload when you edit source files.

### Production Build

To build for deployment:

```bash
npm run build      # Creates optimized production bundle in dist/
npm run preview    # Preview the production build locally
```

The built app in `dist/` can be deployed to any static hosting service (Netlify, Vercel, GitHub Pages, AWS S3, etc.).

## Data Files

The UI uses two pre-computed JSON files located in `src/data/`:

| File | Size | Description |
|------|------|-------------|
| `wardFilterData.json` | 2.8 MB | Ward-level aggregated statistics with 4D breakdown by violation × time × season × day |
| `blockLookup.json` | 2.1 MB | Block-level risk scores and coordinates for address search (24,094 blocks) |

These files are generated by the Python data pipeline (`../data/generate_ward_filter_data.py`) and contain:
- 1,027,421 parking citations aggregated across multiple dimensions
- Bayesian posterior recurrence scores for 8,117,760 block-time combinations
- Census demographics (income, poverty, race) for all 8 wards

**Note**: The UI works standalone with the included JSON files. You only need the `DVA-Data` folder if you want to regenerate these files.

## Key Components

### Map & Navigation
- **WardMap.jsx**: SVG choropleth map with hover/click interactions, uses point-in-polygon for address geocoding
- **Legend.jsx**: Color scale legend and A–F grade reference

### Filtering & State
- **Sidebar.jsx**: Dropdown filters for violation type, time, season, day of week
- **Chip.jsx**: Active filter pills with click-to-remove

### Analytics & Visualization
- **StatCards.jsx**: Summary metrics (total citations, risk score, avg fine, top violation)
- **BottomCharts.jsx**: Hourly distribution, monthly trends, top violations bar chart
- **EquityPanel.jsx**: Three scatter plots examining citations vs. income/poverty/demographics

### Interactive Features
- **AddressSearch.jsx**: Geocodes user address via OpenStreetMap Nominatim API, finds nearest block, displays risk grade
- **DetailPanel.jsx**: Right sidebar showing detailed ward info when a ward is clicked

### Data Layer
- **mockdata.js**: Exports `getFilteredStats()` function that dynamically aggregates ward data based on active filters

## How Filtering Works

The app uses a pre-computed sparse data structure:

```javascript
// Each ward has a breakdown object keyed by:
// "violation|timeBand|season|dayOfWeek" → {c: count, f: fines, b: bayesScore}

breakdown: {
  "FAIL TO DISPLAY A MULTISPACE METER RECEIPT|2pm-4pm|Summer|Fri": {
    c: 1234,    // citation count
    f: 61700,   // total fines
    b: 0.0523   // Bayesian posterior score
  },
  // ... thousands more entries
}
```

When filters change, `getFilteredStats()` iterates through each ward's breakdown, summing only entries matching the active filters. This allows instant updates across all visualizations.

## Equity Analysis

The **Equity Analysis** tab provides three scatter plots:

1. **Citations vs. Median Income** — Are wealthier wards cited more or less?
2. **Citations vs. Poverty Rate** — Does enforcement target high-poverty areas?
3. **Citations vs. % People of Color** — Is there evidence of demographic bias?

Each plot shows:
- **Pearson correlation coefficient** (r) measuring linear relationship strength
- **Per-capita toggle** to normalize by population
- **Risk-colored dots** with one point per ward
- **Disclaimer**: Correlations are descriptive only and don't control for confounders (commercial activity, parking supply, enforcement density)

## Development

To make changes:

1. Edit files in `src/`
2. Vite will hot-reload automatically
3. Check browser console for errors
4. Use React DevTools for debugging

### Common Tasks

**Add a new filter**: Update `DEFAULT_FILTERS` in `mockdata.js`, add UI control in `Sidebar.jsx`

**Change color scheme**: Modify `riskHex()` in `utils/colors.js` and `THEMES` in `utils/theme.js`

**Update data**: Re-run `../data/generate_ward_filter_data.py` to regenerate JSON files (requires DVA-Data folder)

## Deployment

The app is a static site and can be deployed to:

- **Netlify**: `npm run build` → drag `dist/` folder to Netlify
- **Vercel**: Connect to Git repo, auto-builds on push
- **GitHub Pages**: Use `gh-pages` package or GitHub Actions
- **AWS S3 + CloudFront**: Upload `dist/` to S3 bucket configured for static hosting

No server-side configuration needed — just serve the `dist/` folder.
